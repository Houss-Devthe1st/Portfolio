What is the project about?

An online collection of digital information resources, such as e-books, journals, and multimedia content, accessible via the internet. It provides a convenient, location-independent platform for users to access, search, and retrieve information, saving space and costs associated with physical collections.

Project Details

After watching the full demo above, the following blocks will explain the features that this system implements.

Web Design

Using ASP.NET, I developed the website with functional and dynamic front-end and back-end.supports cross-platform development. This provides flexibility in deployment and development environments.The use of C#, a modern and versatile language, along with features like Master Pages and server controls, simplifies development and promotes code reusability. I used the MVC (Model-View-Controller) pattern because it promotes a separation of concerns, making code more organized, maintainable, and scalable.

Styling

To give the website the porfessional look, I used Bootstrap. A popular front-end framework used in web development providing pre-built components (e.g., navigation bars, buttons, forms, cards) and a robust grid system, significantly reducing the amount of manual CSS and HTML coding required. This allows developers to build functional and aesthetically pleasing interfaces more quickly.

In addition to giving the website a polished appearance, Bootstrap was utilized to display the user's logged-in name, making it feel more personalized.

Database

Since it's an E-Library website, it needs to store plenty of information, such as: users accounts, the books and ebooks that will be downloaded or borrowed...So I chose to use MSSQL DB for its robust, enterprise-grade features, deep integration with the Microsoft ecosystem, powerful tools for business intelligence and analytics, and strong security and high availability for mission-critical workloads.

  • - I implemented CRUD operations to add users and books.
  • - The website can identify admins and normal users. the admins have additional tasks available:
    • 1) adding books and users to the database
    • 2) update the status of the borrowed book
    • 3) update the status of a user
    • 4) add authors to the database so that the users can search for books per author name
